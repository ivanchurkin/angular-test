# Angular Test

Тестовое задание для претендента на должность Angular разработчик.

## Структура репозитория

Задание разделено на три уровня:

* first
* second
* third

```
.
├── first/
│   ├── ...
│   └── README.md
│
├── second/
│   ├── ...
│   └── README.md
│
├── third/
│   ├── ...
│   └── README.md
│
└── README.md
```

Каждому уровню соотвествует одноименная директория содержащая README.md с подробным описанием задачи для текущего уровня.

Чем больше уровней вы пройдёте тем лучше.

## Выполнение задания

Делаем fork и/или клонируем:

```
→ git clone git@github.com:ivanchurkin/angular-test.git
```

Переходим в выбранный ур-нь и устанавливаем зависимости:

```
→ cd angular-test/first
→ yarn install // or npm install
```

Запускаем:

```
→ yarn start // npm start
```

Выполняем задание, коммитим и отправляем ссылку на репозиторий.

**Важно!** Постарайтесь сохранить историю разработки в git.

## Проверка решения

При проверке задачи **первого уровня** внимание будет уделено:
* общему пониманию и использованию angular, typescript и rxjs
* семантической разметке html
* применению css стилей
* использованию es6+ возможностей javascript

При проверке задачи **второго уровня** внимание будет уделено:
* использованию angular cdk
* использованию scss и/или bem нотификации
* оптимизации с применением делегирования событий

При проверке задачи **третьего уровня** внимание будет уделено:
* проектированию ngrx store и его использованию

## Обратите внимание

Поскольку каждый уровень подразумевает переиспользование решения из предыдущего уровня, то возможно как поэтапное выполнение, так и итоговое.

Таким образом, при **поэтапном** выполнение вы имеете отдельное решение для каждого уровня. Лучший вариант, если вы сомневаетесь, это позволит оценить решения каждого уровня отдельно.

В случае **итогового** выполннения вы можете объеденить решение нескольких уровней в одно.

## Примечание

Приложения, расположенные в директориях сгенерированы при помощи angular cli 10.2.0:

```
ng new first
  --commit=false
  --inlineStyle=true
  --inlineTemplate=true
  --minimal=true
  --skipGit=true
  --skipTests=true
  --strict=true
  --viewEncapsulation=None
```

Роутинг отсутствует.

В приложение  `first` для стилей используется css, в `second` и `third` используется `scss`.
